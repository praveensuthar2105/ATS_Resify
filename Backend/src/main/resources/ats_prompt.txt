STEP 4 — KNOCKOUT FILTER CHECK (only when JD is provided)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Real ATS systems auto-reject candidates based on hard requirements BEFORE scoring.
If a job description is provided, extract and check these knockout criteria:

1. YEARS OF EXPERIENCE: Extract the minimum years required from the JD (e.g., "5+ years", "3-5 years").
   - Compare against the candidate's total work history duration.
   - If resume shows significantly fewer years (2+ years short), mark as FAIL.
   - For students applying to "entry-level" / "0-2 years" / "new grad" roles: auto-PASS.

2. EDUCATION LEVEL: Extract degree requirement (e.g., "Bachelor's required", "Master's preferred").
   - Check if the resume's Education section meets the requirement.
   - "Preferred" = not a knockout, just a note. "Required" = FAIL if missing.

3. REQUIRED CERTIFICATIONS: Extract any certifications listed as "required" (not "preferred").
   - e.g., "AWS Certified", "PMP", "CPA", "CISSP"
   - FAIL if required certification is not found in the resume.

4. LOCATION / VISA: If JD specifies location requirements or "Must be authorized to work in [country]".
   - Check if resume mentions relevant location or authorization.
   - If resume has no location info, mark as UNKNOWN (not FAIL).

5. MUST-HAVE SKILLS: Extract skills explicitly marked as "required" or "must-have" in the JD.
   - If 50%+ of must-have skills are missing from the resume, mark as FAIL.

For each filter, output: requirement (what the JD asks), status (PASS / FAIL / UNKNOWN), and detail.
If no JD is provided, set knockoutFilters to an empty array.
